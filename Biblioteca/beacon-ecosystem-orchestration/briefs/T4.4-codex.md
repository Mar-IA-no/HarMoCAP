## Task

A browser patchbay served by the Weaver itself (static assets from the FastAPI app, no build step, no framework — vanilla JS + WebSocket). Criterion: patching must feel like performance, not like filling a form.

1. Serve a single-page patchbay at `/` (or `/patchbay`) from the existing app:
   - left: sources with their channels (from registry/state);
   - right: instruments with their capabilities;
   - center: the route list for the active scene — create, enable/disable, edit transform chain (scale_range/curve/smoothing/gate/combine with their declared fields), delete;
   - scene selector + hot switch; global PANIC button (latched, visually unmistakable, with clear/recover requiring the panic_generation flow from the contract).
2. All mutations through the Stage WS protocol with expected_stage_revision handling; live updates from state.event broadcasts (registry changes, route changes, panic state). Reconnect with backoff.
3. Route creation should support the common case fast: pick source channel → pick instrument capability (+bindings) → route appears. Transform editing available but not mandatory for a basic patch.
4. Minimal, dark, readable styling. No external CDNs (must work offline on the event LAN).
