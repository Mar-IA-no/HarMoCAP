## Task

Make beacon-spatial a formally patchable instrument, in three parts:

### 1. `beacon_spatial.contract.json` (+ golden sidecar)

Formalize the 69 OSCdefs of `beacon.scd` (the REAL current engine — read it; the docs were just aligned to it) into the Instrument Control v1 template: namespace `/beacon/*`, all addresses with arg types/ranges/N bounds (gain/az/dist/solo N=1..13, q N=1..12, mix/master, record/*, reset), handshake section, state_sync section (see part 2), receiver rules. No voice_model_alias section — beacon-spatial is the canonical counterexample (gain/az/dist/Q/mix over 13 bands of a live analog signal). Generate `contract_id` with the copied codec and write the golden sidecar.

### 2. Bidirectional state in `beacon.scd`

Implement the state_sync declared in the manifest, following the template's dump mode:
- An OSC responder for `/beacon/state` (request) that emits an atomic dump: `/beacon/state/begin` → current value of every readable address (all 13 bands' gain/az/dist/solo/q + mix + master) → `/beacon/state/end`.
- Keep it minimal and robust: read current values from the existing control structures (the .scd already stores them — find how; do not duplicate state).

### 3. Configurable listen host/port in `beacon.scd`

Today addresses/ports are hardcoded (127.0.0.1, sclang OSC on 57120, scsynth on 57110). Make the OSC listen port and bind address configurable via command-line args and/or environment variables (pick the mechanism `start-beacon.sh` can pass through; keep defaults exactly as today so nothing existing breaks). Update `start-beacon.sh` ONLY if needed to pass the flags through (backward compatible).

### 4. Round-trip test

`tests/test_osc_contract.py` (pytest, stdlib only + python-osc if already in venv — prefer stdlib codec): manifest validates + contract_id matches sidecar; a simulated dump bundle round-trips through the codec; address table in the manifest covers every OSCdef found in `beacon.scd` (write a small extractor: parse `OSCdef` lines and diff against manifest addresses).
