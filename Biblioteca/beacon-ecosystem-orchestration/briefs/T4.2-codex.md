## Task

Implement the minimum complete, headless Weaver MVP described in CORE_DESIGN:

1. `src/harmonic_weaver/engine/` package (or equally clear package layout) containing:
   - immutable stage state + serialized mutations/revisions;
   - source and instrument registries with contract_id / stream_id gating;
   - declarative route compiler and evaluator: `scale_range`, `curve`, `smoothing`, `gate`, `combine`; finite-value/range validation; `observed|held|invalid` policy; destination collision detection;
   - derived source aggregators (fixed_hz and on_input) with declared validity/count rules;
   - scenes, atomic route.batch, and hot switch policies from CORE_DESIGN;
   - global latched panic state and per-instrument safety-profile dispatch. Provide an in-memory/recording output transport suitable for tests; do not invent live OSC behavior outside a declared Instrument Control capability.

2. Headless FastAPI server with the Stage Contract WebSocket protocol. No UI. Implement server/client hello contract gate, `state.subscribe` snapshot/events, command ack/error, route CRUD/batch, scene upsert/switch, panic trigger/clear, and registry events. It must be runnable through `python -m harmonic_weaver.weaver` (or a documented `harmonic-weaver-weaver` entry point). Update pyproject only if required.

3. Adapter seams:
   - accept source frames from the existing drivers directly via their callback contract;
   - accept installed manifests and a testable instrument-send callback. In this task, a test recorder is enough; live beacon/shaper OSC adapters follow only if the manifest capability shape already makes them trivial and testable.

4. Evidence/reporting:
   - `reports/<run_id>/` writer containing canonical scene snapshot, accepted contract IDs, state/behavior events, and latency summary stubs measured from this engine (not invented claims).
