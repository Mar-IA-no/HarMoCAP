## Root cause (verified, not hypothesis)

Every route evaluation ended in a transport error (`route_evaluations=39736`, `transport_errors=39736`, `instrument_writes=0`). Captured exception:

```text
TypeError: cannot pickle 'mappingproxy' object
record: instrument=shaper capability=harmonic_gain address=/digital/harmonic/1/gain value=0.0489...
```

Mechanism: `rehearsal/support.py` `LiveOSCTransport._send` calls `dataclasses.asdict(record)`. The engine constructs `OutputRecord` with `bindings=freeze(destination.definition["bindings"])` — a `types.MappingProxyType`. `asdict` deep-copies fields via pickle; mappingproxy cannot be pickled → TypeError raised AFTER `sock.sendto`, before the audit append. Consequences: audit file never written, metrics lie (writes counted as errors), downstream assertions starve. The existing unit test passed only because it builds OutputRecord with a plain dict.
